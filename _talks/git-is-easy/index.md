## Abstract

This has been a good year for Git. More and more open source projects --
including perl5 and rakudo perl6 -- are moving their source code to Git
repositories, and "patches welcome" is slowly being replaced with "tell me
where to pull from."

There's been some loose talk suggesting that Git is highly complex and has a
bizarre interface that makes it difficult to learn. This is completely and
entirely false, mostly.

This talk will explain how to get started with git, how it works, and how to
think about git in order to really know what you're doing.
